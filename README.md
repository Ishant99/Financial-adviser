# FinAdvisor — Personal AI Financial Advisor

A personal CFO that knows my entire financial life — mutual funds, bank accounts, EPF, credit cards, goals — and produces a specific, numbered, actionable money plan on the 1st of every month.

Built for one user (myself) first. If it works for me, it will work for others.

---

## What It Does

| Feature | Description |
|---|---|
| **Monthly Money Plan** | AI-generated plan with Actions, Watches, and Wins. Real numbers from the rules engine; Claude writes the narrative. |
| **Net Worth Dashboard** | Total net worth + category breakdown, updated on demand. |
| **Portfolio Analytics** | Asset allocation by type, concentration risk, per-holding CAGR. |
| **SIP Health** | Every SIP with XIRR (Newton-Raphson, daily background job), linked goals. |
| **Goal Tracker** | Monte Carlo probability of success (10k paths, NumPy), P10/P50/P90 corpus projections. |
| **Cash Flow** | Rolling N-month view of income vs expenses, category drill-down, grouped bar chart. |
| **Tax Summary** | Current-FY LTCG/STCG estimates, Section 80C SIP tracking. |
| **CAS Upload** | Import mutual fund holdings from CAMS/Karvy CAS PDFs via `casparser`. |
| **AI Recommendations** | Context-aware portfolio recommendations generated by Claude with `tool_use`. |

## Architecture

```
Browser (Next.js 16)
    │  Axios + TanStack Query
    ▼
.NET 10 API  ←── Clean Architecture (Domain → Application → Infrastructure → Api)
    │  EF Core      │  HttpClient + Polly
    ▼              ▼
PostgreSQL 16    Python FastAPI analytics service
                     ├── /cas/parse          (casparser)
                     ├── /simulation/…       (NumPy Monte Carlo)
                     ├── /xirr/compute       (Newton-Raphson)
                     ├── /recommendations/…  (Claude tool_use)
                     └── /plan/generate      (Claude tool_use)
                                │
                         Anthropic Claude
                         claude-haiku-4-5
```

**Core principle:** The LLM writes words, not numbers. All figures come from the rules engine. Claude is instructed via system prompt: *"Use ONLY figures present in the data. Do NOT invent any numbers."*

See [`docs/architecture.md`](docs/architecture.md) for the full architecture document.

## Quick Start

### One command (recommended for development)

From the project root (`Financial-adviser/`):

```powershell
.\dev.cmd
```

That single command:

1. Loads a root `.env` (for `ANTHROPIC_API_KEY`) if present.
2. Installs the launcher's one dependency (`concurrently`) the first time only.
3. Starts Docker Desktop if it isn't already running, then brings up **Postgres** and waits until it's healthy.
4. Runs the **.NET API** (`:5050`), **Python analytics** (`:8000`) and **Next.js web app** (`:3000`) together with hot-reload, in one terminal with colour-prefixed logs.

Press **Ctrl+C** once to stop all services (Postgres is stopped automatically too).

> Open **http://localhost:3000**. First run is slower (NuGet restore + first Next build); subsequent runs are fast.
>
> Equivalent under the hood: `npm run dev` (the `dev` script in the root `package.json`). On macOS/Linux run `pwsh ./dev.ps1`, or just `npm run dev` after `docker compose up -d postgres`.

### Full containerised run (production-like)

```bash
cp .env.example .env      # set ANTHROPIC_API_KEY
docker compose up --build # everything in containers; open http://localhost:3000
```

### Manual (without the launcher)

```bash
# Terminal 1 — Postgres
docker compose up -d postgres

# Terminal 2 — Apply EF migrations
dotnet ef database update \
  --project FinAdvisor/src/FinAdvisor.Infrastructure \
  --startup-project FinAdvisor/src/FinAdvisor.Api

# Terminal 3 — .NET API (port 5000)
dotnet run --project FinAdvisor/src/FinAdvisor.Api

# Terminal 4 — Python analytics service (port 8000)
cd src/analytics-service
pip install -e .
uvicorn app.main:app --reload --port 8000

# Terminal 5 — Next.js dev server (port 3000)
cd src/web
npm install
cp .env.local.example .env.local
npm run dev
```

## Working Principles

- Every AI-generated line is read and understood before committing — no blind AI commits
- Financial math is verified against a known case before shipping
- The LLM writes words, not numbers — all figures come from the rules engine
- Conservative advice always: emergency fund before investing, debt before returns
- Every sprint ends with a working, deployable state (`dotnet build` green, `tsc --noEmit` zero errors)
